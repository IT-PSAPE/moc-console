# A domain-agnostic, composable Timeline primitive

The existing cue-sheet timeline (`apps/console/src/components/timeline`) is good but coupled: `TimelineRoot` hard-codes its sidebar/canvas, threads a `readOnly` boolean through context, and bakes a Supabase controller/follower clock into the component. We need the same surface for two unrelated domains — Cue sheets (events) and the Broadcasts section (playlists) — so we are extracting a single domain-agnostic **Timeline** primitive into `@moc/ui`. It speaks only **Lane** and **Block** (see CONTEXT.md); each domain maps its own `Track`/`Cue` onto those and renders Block contents as real children. The domain `Cue` types are deliberately *not* unified.

## Considered options

- **Unify on one shared `Cue` model.** Rejected: it bloats the type with mutually-exclusive fields (cue-sheet `type`/`startMin` vs media reference) and couples two bounded areas the glossary keeps separate. The primitive needs neither model — it needs `{id, start, duration}` per Block.
- **Build the primitive on the cue-sheet `Track`/`Cue` shape** and adapt playlists to it. Rejected: bends the playlist domain to fit the event domain; the abstraction should be below both, not one of them.
- **Headless collection parts that auto-iterate** (`<Timeline.Lanes>` reading the array from context, one template child). Rejected: the single template child is effectively a per-item render prop — the pattern [feedback_composable_components] explicitly avoids. The consumer maps data → parts explicitly in JSX instead; the tree documents part *types and nesting*, not every datum.
- **A `readOnly` prop / a capabilities object in context.** Rejected: boolean-driven conditional rendering, which the composition rules forbid. Editing affordances are instead composable parts (`Block.ResizeHandle`, `Block.DragHandle`, `Timeline.AddLane`); read-only is literally a smaller tree. `Timeline.Playhead` (scrubbable) and `Timeline.PlayheadMarker` (inert) are distinct parts rather than one part behind a flag.
- **Primitive owns a generic clock; media slaves to it** (or a clock/media mode switch). Rejected: a generic ticker fights the browser — media buffering/latency drifts the visual playhead from real playback — and a mode switch re-bakes both domains into the primitive. The time source is an **injected Transport** instead: the cue-sheet supplies a clock transport (ticker + controller/follower sync), the playlist supplies a media transport (the playing media *is* the clock).
- **Keep the primitive controlled (`tracks` in, `onChange` out)** as today. Rejected for the child-mutation story: children would depend on consumer wiring. The provider owns the live working copy and exposes mutators on context; committed changes leave through an injected persist port (same shape as the Transport port). The consumer seeds initial data and decides debounce/optimistic/DB-sync outside the primitive.

## Consequences

- **`@moc/ui` gains a pure primitive.** The Supabase-coupled `use-playback-sync` does not move with it; it becomes a console-side ClockTransport adapter. The primitive imports no `@moc/data`. Follower state arrives *through* the ClockTransport, not as a prop, so the controller/follower model is preserved without coupling.
- **Two injected ports at the root:** `transport` (time in) and a persist port (committed mutations out). Everything else — zoom, drag, selection, playhead position, working data — is provider-owned context state; no prop drilling below the root.
- **The tree is the documentation.** `Timeline` → `Timeline.Toolbar` / `Timeline.Ruler` / `Timeline.Sidebar` (`Timeline.LaneHeader`) / `Timeline.Canvas` (`Timeline.Lane` → `Timeline.Block` → `Block.*` handles; `Timeline.Playhead`/`PlayheadMarker`; `Timeline.Preview`). Presence = rendered; order = layout/stack order.
- **`Timeline.Preview` is an empty composition slot** in the primitive — the domain fills it (see ADR-0004 for the playlist compositor). Cue sheets simply omit it.
- **Two existing consumers migrate onto it:** the editable cue-sheet timeline and the read-only `event-timeline.tsx` (proving the editable and inert trees), with no schema change on the cue-sheet side.
